## 一、项目概述

### 核心定位
**餐饮门店跨平台内容运营工具**：用AI替代人力，快速将门店素材（一图、一视频、一海报）转化为小红书、大众点评、抖音三平台适配内容。

### 目标用户
- **有自营门店的餐饮品牌**（2家门店，当前试水）
- 痛点：店员不会写、没时间写、发出去没人看
- 诉求：解放人力 + 提升发布频率 + 统一品牌形象

### 竞对范围（v2.0调整）
| 平台 | 典型工具 |
|------|---------|
| 小红书 | 月溪、羚轨、即时工具 |
| 大众点评 | 商迪sar、店小秘 |
| 抖音 | 巨量引擎、创意助手 |
| 通用 | Notion AI内容工厂 |

> ⚠️ 不做Facebook/Instagram，等国内三平台验证完成再说。

## 二、OCR方案（v2.0调整）

### 起步策略：免费优先，快速验证

#### 方案A：微信扫一扫 OCR（推荐启动）
- **优点**：用户已有微信，零成本，文字识别效果好
- **缺点**：需人工介入截图
- **适用场景**：门店员工拍图 → 微信识别 → 粘贴文字

#### 方案B：百度OCR免费额度
- **额度**：每天500次免费（通用文字识别）
- **接入**：Simple OCR / PaddleOCR 开源版
- **适用场景**：API自动化流程

#### 方案C：PaddleOCR（自部署，开源免费）
- **优点**：完全免费，可本地部署
- **缺点**：需服务器资源
- **适用场景**：后期量大后自建服务

### 演进路径
```
阶段1（0-1万次/月）：微信扫一扫 + 百度OCR免费额度
阶段2（1-10万次/月）：升级百度OCR付费 + PaddleOCR自部署
阶段3（>10万次/月）：商用OCR方案（阿里云、腾讯云）
```

## 四、三平台功能优先级（v2.0调整）

### 平台特性对比

| 维度 | 小红书 | 大众点评 | 抖音 |
|------|--------|---------|------|
| 内容形式 | 图文/短视频 | 图文/团购 | 短视频/直播 |
| 核心价值 | 攻略、口碑 | 评分、团购 | 曝光、引流 |
| 内容风格 | 生活化、真实感 | 正式、专业 | 短平快、娱乐化 |
| 痛点 | 写不出"种草感" | 不知道怎么写评价 | 不会拍视频/写脚本 |

### 功能优先级

#### 🔴 P0（第一版必须）
1. **内容提取** - 从截图/链接提取门店名称、菜品、评价
2. **小红书适配** - 生成种草文案 + emoji + 话题标签
3. **大众点评适配** - 生成评价引导文案 + 团购描述
4. **抖音适配** - 生成短视频脚本/口播文案

#### 🟡 P1（第二版）
5. **多图拼接** - 自动拼接9宫格
6. **评论管理** - 跨平台评论汇总
7. **定时发布** - 草稿箱 + 定时发送

#### 🟢 P2（第三版）
8. **数据看板** - 点赞/收藏/转化统计
9. **AI生成图片** - 根据文案配图
10. **门店矩阵管理** - 多门店统一管理

## 六、技术架构（保持不变）

### 核心流程
```
截图/链接 → OCR识别 → AI改写 → 三平台适配 → 草稿/发布
```

### 技术选型
| 模块 | 技术 | 说明 |
|------|------|------|
| OCR | 微信扫一扫/百度OCR/PaddleOCR | 免费方案起步 |
| AI改写 | GPT-4o / Claude | 多风格文案生成 |
| 图片处理 | Pili / Canva API | 9宫格拼接 |
| 存储 | 微信小程序云开发 / Supabase | 草稿+历史记录 |
