# OpenClaw Gateway 断连监测与自动恢复机制

## 1. 需求概述

- **目标**：监测 OpenClaw Gateway 服务状态，检测到断连后自动尝试恢复，并记录日志
- **触发条件**：Gateway 服务停止/崩溃
- **日志位置**：`D:\OpenClaw\断连日志\`

## 2. 系统架构

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   Cron Job     │ ──> │  Gateway Monitor │ ──> │   日志系统      │
│  (定期检测)     │     │  (检测+恢复)      │     │ (D:\OpenClaw\) │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

## 3. 功能设计

### 3.1 检测机制

| 项目 | 内容 |
|------|------|
| 检测方式 | `openclaw gateway status` |
| 检测间隔 | 每 2 分钟执行一次 |
| 判定标准 | 返回状态非 "running" 视为断连 |

### 3.2 恢复策略

| 项目 | 内容 |
|------|------|
| 恢复命令 | `openclaw gateway restart` |
| 最大重试次数 | 2 次 |
| 重试间隔 | 30 秒 |
| 恢复成功判定 | `openclaw gateway status` 返回 "running" |

### 3.3 日志格式

```
================================================================================
[2026-03-20 07:35:22] ====== Gateway 断连检测 ======
[检测结果] 服务已停止
[修复尝试] 第1次 - 开始恢复...
[修复结果] 第1次 - 成功
[最终状态] 已恢复
================================================================================
```

### 3.4 日志文件

- **路径**：`D:\OpenClaw\断连日志\`
- **命名**：`gateway-monitor-YYYY-MM-DD.log`
- **编码**：UTF-8
- **滚动**：按天分隔，自动清理（可选）

## 4. 实施计划

1. 创建日志目录
2. 编写 PowerShell 监测脚本
3. 配置 cron job 定期执行
4. 测试验证

## 5. 风险与限制

- 需确保 `openclaw` 命令在系统 PATH 中
- 若服务因系统级问题崩溃，恢复可能无效
- 日志目录需有写入权限
