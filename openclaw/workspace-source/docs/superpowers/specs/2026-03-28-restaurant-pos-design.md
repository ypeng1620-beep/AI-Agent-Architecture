# 餐饮收银系统规格文档

**项目名称**: 餐饮收银系统 (Restaurant POS)  
**版本**: 1.0  
**日期**: 2026-03-28

---

## 1. 项目概述

- **项目类型**: Windows 桌面端餐饮收银管理系统
- **核心功能**: 扫码点餐、收银管理、第三方订单聚合、会员管理
- **目标用户**: 单门店餐饮商家（日均20桌）
- **部署方式**: Windows 本地部署

---

## 2. 技术栈

| 层级 | 技术选择 | 说明 |
|------|----------|------|
| 后端 | Java (Spring Boot 3.x) | 稳定，第三方集成丰富 |
| 前端桌面端 | Electron + Vue 3 | 跨平台桌面应用 |
| 前端管理后台 | Vue 3 + Element Plus | Web管理界面 |
| 数据库 | MySQL 8.0 | 主数据库 |
| 缓存 | Redis | 会话/缓存 |
| 构建工具 | Maven + Vite | 前后端构建 |

## 4. 功能模块

### 4.1 扫码点餐模块

| 功能 | 描述 |
|------|------|
| 桌码生成 | 每桌唯一二维码，绑定桌位信息 |
| 菜单展示 | 分类展示菜品，支持图片/规格/口味 |
| 购物车 | 加菜/减菜/备注/规格选择 |
| 下单 | 提交订单到后厨打印/叫号 |
| 支付 | 微信/支付宝扫码支付 |
| 订单查询 | 顾客可查看历史订单 |

### 4.2 收银管理模块

| 功能 | 描述 |
|------|------|
| 开台/并台/转台 | 桌位状态管理 |
| 点餐 | 直接在收银机上手动点餐 |
| 加菜 | 订单追加菜品 |
| 改单 | 修改数量/口味/备注 |
| 结账 | 支持现金/微信/支付宝/会员卡 |
| 小票打印 | 打印结账小票 |
| 退菜/退款 | 退款操作及日志 |

### 4.3 订单聚合模块

| 平台 | 对接方式 |
|------|----------|
| 美团 | 开放平台API（订单拉取/接单/拒单） |
| 饿了么 | 开放平台API（订单拉取/接单/拒单） |
| 抖音 | 小程序API（订单拉取/接单） |
| 聚合展示 | 统一订单列表，按平台分类 |

**对接功能**:
- 自动拉取新订单
- 接单/拒单/取消操作
- 订单备注同步
- 打印对接

### 4.4 会员系统模块

| 功能 | 描述 |
|------|------|
| 会员注册 | 手机号注册 |
| 会员卡 | 储值卡/折扣卡 |
| 积分 | 消费积分累计/抵扣 |
| 优惠券 | 满减券/折扣券 |
| 会员等级 | 按消费额自动升级 |

### 4.5 菜品管理模块

| 功能 | 描述 |
|------|------|
| 菜品库 | 菜品名称/图片/价格/规格 |
| 分类管理 | 大类/子类 |
| 上下架 | 快速启用/停售 |
| 规格/口味 | 规格(如大/小)、口味(如辣度) |
| 估清 | 售罄标记 |

### 4.6 报表中心模块

| 报表 | 描述 |
|------|------|
| 日结报表 | 当日营收/订单/客单价 |
| 月度报表 | 月度趋势分析 |
| 菜品销售排行 | 热销/滞销菜品 |
| 平台订单统计 | 各平台订单量/营收 |
| 会员分析 | 新增/活跃/消费TOP |

## 5. 数据模型

### 5.1 核心实体

```
Tables:
├──菜品 (dish)
├──分类 (category)
├──桌位 (table)
├──订单 (order)
├──订单明细 (order_item)
├──支付 (payment)
├──会员 (member)
├──会员卡 (member_card)
├──优惠券 (coupon)
└──第三方订单 (third_party_order)
```

### 5.2 API 设计

| 模块 | API 路径 | 方法 |
|------|----------|------|
| 菜品 | /api/dish | GET/POST/PUT/DELETE |
| 订单 | /api/order | GET/POST/PUT |
| 桌位 | /api/table | GET/POST/PUT |
| 会员 | /api/member | GET/POST/PUT |
| 支付 | /api/payment | POST |
| 第三方 | /api/third-party/{platform}/order | GET/POST |

## 7. 部署架构

### 7.1 硬件要求

| 组件 | 配置 |
|------|------|
| 服务器 | CPU 4核+ / 内存 8G+ / SSD 256G+ |
| 收银机 | 普通PC / POS一体机 |
| 打印机 | 58mm 热敏小票打印机 (USB/蓝牙) |
| 二维码 | 桌位二维码标牌 |

### 7.2 软件部署

1. **MySQL**: 安装包/容器 → 创建数据库 → 执行SQL
2. **Redis**: 安装包/容器
3. **后端**: JAR包部署 → 启动脚本
4. **前端**: Electron打包 → 安装到收银机
